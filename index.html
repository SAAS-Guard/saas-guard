About the Project

What is SAAS-Guard?

A multi-model AI framework for adaptive, explainable access control in cloud-native SaaS environments.

Modern SaaS platforms face increasing challenges in managing user access securely and adaptively within cloud-native, multi-tenant environments. Traditional access control mechanisms such as RBAC and ABAC rely heavily on static rules and predefined user-role mappings.

These models struggle to adapt to dynamic user behavior, contextual changes, or sophisticated insider threats — leading to over-privileged access, policy misconfigurations, and delayed threat detection.

SAAS-Guard proposes an AI-powered framework that continuously learns from user behavior, detects anomalies in real time, predicts access decisions, and provides transparent, auditable justifications — advancing Zero Trust security and GDPR compliance.

Research Domain

Architecture Design

The SAAS-Guard system architecture integrates four AI-driven modules into a unified, modular pipeline for adaptive access control in SaaS environments. The architecture follows a microservice-oriented design with:

  • Data Ingestion Layer: Collects user behavioral logs, access requests, and contextual metadata from SaaS applications.
  • Anomaly Detection Module: Processes behavioral streams through unsupervised ML models (Isolation Forest, Autoencoders) for real-time insider threat detection.
  • Role Recommendation Engine: Applies behavioral clustering (DBSCAN, K-Means) to suggest least-privilege role assignments.
  • Access Classifier: Uses ensemble ML models(SBERT, Isolation Forest) for Intent aware Risk-based dynamic access decisions with confidence scoring.
  • Explainability Layer: Integrates Post-Hoc XAI Techniques and a constrained local LLM to generate human-readable, audit-ready decision justifications.
  • Policy Enforcement Gateway: Enforces decisions in real-time with Zero Trust architecture principles.

Literature Survey

The literature highlights critical challenges in SaaS access control, including the limitations of static RBAC/ABAC models in dynamic cloud environments. Key findings include:

  • Traditional access control models fail to adapt to evolving user behavior and contextual changes in multi-tenant SaaS platforms.
  • Existing insider threat detection systems rely heavily on labeled datasets, limiting their effectiveness against novel attack vectors.
  • Role mining approaches often produce over-privileged assignments due to insufficient behavioral analysis.
  • Current ML-based access decision systems lack transparency, making audit compliance difficult.
  • Explainable AI (XAI) research has not been adequately applied to access control decision pipelines.
  • Few systems integrate anomaly detection, role recommendation, access classification, and explainability into a unified framework.

Research Gap

The core research challenge lies in integrating unsupervised anomaly detection, behavioral role recommendation, ML-based access decisioning, and explainable AI into a unified, modular architecture.

Unlike conventional access control, existing frameworks fail to combine these capabilities into a single, adaptive, and interpretable system. There is a significant absence of systems that simultaneously address dynamic insider threats, least-privilege enforcement, real-time access classification, and audit-ready explainability within one pipeline.

Research Problem

How can a cybersecurity system provide mathematically sound, model-specific, and empirically validated explanations for each component of a multi-stage AI pipeline — while synthesizing diverse explanations into concise, actionable natural language summaries that prevent generative AI hallucination?

Research Objectives

  1. Develop an unsupervised anomaly detection system that identifies insider threats and zero-day attacks in real time without labeled training data.
  2. Design a behavioral clustering-based role recommendation engine that enforces least-privilege access based on actual user activity patterns.
  3. Build an ML-based access classifier that dynamically predicts allow/deny decisions with risk-calibrated confidence scores.
  4. Create an explainability engine using Post-Hoc XAI Techniques and a constrained local LLM that generates forensic-quality, hallucination-free decision summaries.
  5. Integrate all four components into a unified, modular framework that advances Zero Trust security and GDPR compliance for SaaS platforms.

Methodology

This study employs a mixed-methods design science research approach:

  • Anomaly Detection: Unsupervised learning using Isolation Forest and Autoencoder neural networks trained on user behavioral logs for real-time insider threat detection.
  • Role Recommendation: Behavioral clustering using DBSCAN and K-Means algorithms with peer-group similarity analysis for dynamic least-privilege role suggestions.
  • Access Classification: Ensemble ML models (SBERT and Isolation Forest) trained on contextual access log features for taking intent aware risk based access decisions
  • Explainability: Post-hoc interpretability analysis using Post-Hoc XAI Techniques, combined with a constrained local LLM (fine-tuned with guardrails) for generating natural language decision summaries.
  • Validation: Performance evaluation through accuracy, precision, recall, F1-score metrics, along with security testing and user studies for explainability quality.
